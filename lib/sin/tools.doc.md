# `lib/sin/tools.ts`

Single source of truth for the sin-code Go backend surface.

## What it does

Exports the canonical list of 32 `sin-code` subcommands and 44 `sin_*` MCP tools
that the WebUI may expose to the chat model. Every route handler, prompt, and
status tile imports from this file so the UI never drifts from the Go binary.

## Dependencies

- Generated by hand from `SIN-Code-Bundle/cmd/sin-code/main.go` and
  `cmd/sin-code/internal/serve.go`.
- Imported by `lib/sin/client.ts`, `lib/sin/run.ts`, `lib/sin/mcp.ts`,
  `app/api/chat/route.ts`, `app/api/sin/status/route.ts`, and
  `components/sin-status-tile.tsx`.

## Important values

- `SIN_CODE_SUBCOMMANDS` — 32 whitelisted CLI subcommands (v2.5.0).
- `SIN_MCP_TOOLS` — 44 MCP tool names exposed by `sin-code serve`.
- `SIN_CODE_INSTALL_CMD` — one-liner to install the Go binary.

## Why these decisions

Historically the tool lists were hand-maintained in multiple files, causing
Issues #1-#5. Centralizing here prevents divergence.

## Caveats

When the Go binary gains a new subcommand or MCP tool, this file must be updated
first. Counts like "32" or "44" should never be hardcoded elsewhere; use
`SIN_CODE_SUBCOMMANDS.length` and `SIN_MCP_TOOLS.length` instead.
